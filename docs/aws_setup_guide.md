# AWS Free Tier & IAM Security Setup Guide

This guide walks you through setting up a secure AWS Cloud environment from scratch. We will create a free sandbox account, secure it with MFA, create an IAM scanning user with Read-Only privileges, and configure credentials for CloudLockr.

---

## 1. Create an AWS Free Tier Account
AWS provides a 12-month Free Tier sandbox which includes free monthly allowances for EC2, S3, RDS, and other services.

1. Go to the [AWS Free Tier Sign-up Portal](https://aws.amazon.com/free/).
2. Click **Create an AWS Account**.
3. Enter your email, set a password, and choose an AWS Account Name (e.g., `cloudlockr-lab`).
4. Complete the contact details. Choose **Personal** for account type.
5. Enter credit/debit card details. AWS charges a temporary $1 authorization to verify your identity (refunded within 3-5 days).
6. Verify your identity via SMS/Phone verification.
7. Choose the **Basic Support (Free)** plan and complete registration.

---

## 2. Secure Your Root Account (Crucial!)
The Root Account (the email you signed up with) has unrestricted access to all billing and resource operations. It should **never** be used for daily tasks or programmatic API connections.

1. Sign in to the [AWS Management Console](https://console.aws.amazon.com) using your root user email.
2. In the top-right corner, click your Account name and select **Security Credentials**.
3. Under **Multi-factor authentication (MFA)**, click **Assign MFA device**.
4. Choose **Authenticator app** (e.g., Google Authenticator, Authy) and click **Next**.
5. Click **Show QR code** and scan it with your mobile authenticator app.
6. Enter two consecutive codes generated by the app and click **Add MFA**.
7. Close the page. Log out of the root user account and log in again to verify MFA is active.

---

## 3. Create an IAM User for Scanning
We will create an Identity and Access Management (IAM) user specifically for our CloudLockr scanner. This user will only have read privileges to audit configurations.

1. Search for **IAM** in the top AWS search console and open it.
2. In the left navigation sidebar, click **Users** under *Access management*.
3. Click **Create user**.
4. Enter User details:
   * **User name**: `cloudlockr-scanner`
   * Leave the *Provide user access to the AWS Management Console* checkbox **unchecked** (this scanner only needs programmatic API access, not console password login).
   * Click **Next**.
5. Set Permissions:
   * Select **Attach policies directly**.
   * In the policy search box, type `ReadOnlyAccess`.
   * Check the checkbox next to **ReadOnlyAccess** (AWS Managed Policy).
   * Click **Next**.
6. Review configurations and click **Create user**.

---

## 4. Generate Programmatic Access Keys
To authorize the CloudLockr Node.js backend to query AWS resources, we need access keys.

1. In the IAM Users console, click on the newly created user `cloudlockr-scanner`.
2. Select the **Security credentials** tab.
3. Scroll down to the **Access keys** section and click **Create access key**.
4. On the *Access key best practices* step, select **Application running outside AWS** and click **Next**.
5. (Optional) Enter a description tag like `CloudLockr Scanner CLI Access` and click **Create access key**.
6. **Save your credentials:**
   * Copy the **Access key ID** (e.g., `AKIAIOSFODNN7EXAMPLE`).
   * Click **Show** next to the **Secret access key** and copy it.
   * Click **Download .csv file** to save these key files locally.
   * **CRITICAL WARNING:** Keep these keys secret. Never commit them to Git, email them, or upload them anywhere.

---

## 5. Install & Configure the AWS CLI (Optional)
The AWS Command Line Interface (CLI) is useful for testing credentials directly on your developer machine.

1. Download and install the AWS CLI:
   * **Windows:** Download the [64-bit MSI installer](https://awscli.amazonaws.com/AWSCLIV2.msi) and install it.
   * **macOS:** Run `curl "https://awscli.amazonaws.com/AWSCLIV2.pkg" -o "AWSCLIV2.pkg"` and install the package.
2. Open your terminal/PowerShell and run:
   ```bash
   aws --version
   ```
3. Run the configuration wizard:
   ```bash
   aws configure
   ```
4. Enter your generated access details when prompted:
   * **AWS Access Key ID**: Paste your `Access key ID`
   * **AWS Secret Access Key**: Paste your `Secret access key`
   * **Default region name**: `us-east-1` (or your preferred region)
   * **Default output format**: `json`
